# Lessons Learned — SODV2251 Project 1

## Git workflow

- **Initial setup**: Cloned an empty GitHub repo first, then scaffolded the Expo project inside it. This is cleaner than `git init` locally and adding the remote after, because the remote is already the source of truth from the start.

- **`create-expo-app` inside an existing repo**: The tool detected the parent `.git` folder and prompted whether to skip initializing a new git repo. Answering yes keeps a single repo rather than a nested one.
